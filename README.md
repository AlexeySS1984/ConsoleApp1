# Практическая работа №7 — Отладка программы различными способами

> **Цель:** произвести процедуру отладки программного обеспечения встроенными средствами среды программирования Microsoft Visual Studio.

---

## Задание 1 — Консольное приложение «Фибоначчи»

**Источник:** [Microsoft Learn — Use Visual Studio Debugger](https://learn.microsoft.com/ru-ru/training/modules/dotnet-debug-visual-studio/4-use-visual-studio-debugger)  
**Ветвь Git:** `master`  

### Описание задачи

Приложение вычисляет числа Фибоначчи. В ходе работы была найдена и исправлена логическая ошибка в рекурсивной функции (неверное базовое условие выхода), из-за которой при n = 0 возвращалось некорректное значение.

### Использованные способы отладки Microsoft Visual Studio

| № | Инструмент | Применение |
|---|---|---|
| 1 | **Точки останова (Breakpoints)** | Установлены на входе в рекурсивную функцию `Fibonacci()` и на операторе `return`, чтобы отслеживать каждый вызов и промежуточные результаты. |
| 2 | **Условные точки останова (Conditional Breakpoints)** | Настроено условие `n == 0` для остановки только при достижении базового случая рекурсии. |
| 3 | **Шаг с заходом — Step Into (F11)** | Использовался для пошагового входа внутрь каждого рекурсивного вызова `Fibonacci()`, чтобы наблюдать изменение параметра `n`. |
| 4 | **Шаг — Step Over (F10)** | Применялся для выполнения строк без захода внутрь вызовов в методе `Main`, чтобы быстро перемещаться по коду верхнего уровня. |
| 5 | **Окно локальных переменных (Locals Window)** | Отслеживались текущие значения `n` и `result` на каждом уровне стека. |
| 6 | **Окно контрольных значений (Watch Window)** | Добавлены выражения `n`, `Fibonacci(n-1)` для постоянного наблюдения за значениями между вызовами. |
| 7 | **Подсказки по данным (Data Tips)** | При наведении мыши на переменные прямо в редакторе отображались текущие значения в точке остановки. |
| 8 | **Стек вызовов (Call Stack Window)** | Просмотрена цепочка рекурсивных вызовов `Fibonacci(6) → Fibonacci(5) → ... → Fibonacci(0)` для анализа глубины рекурсии. |

### XML-комментарии

Добавлены XML-комментарии к методам `Main` и `Fibonacci` (см. файл `fibonacci_Program.cs`).

## Задание 2 — Консольное приложение «Галактики»

**Источник:** [Microsoft Learn — Debugging for Absolute Beginners](https://learn.microsoft.com/ru-ru/visualstudio/debugger/debugging-absolute-beginners?view=vs-2022&source=recommendations&tabs=csharp)  
**Ветвь Git:** `galaxies-debug`

### Описание задачи

Приложение выводит список галактик по типу. Ошибка заключалась в некорректном сравнении строк при фильтрации объектов по полю `GalaxyType` — использовалось сравнение без учёта регистра, что приводило к тому, что ни одна галактика не выводилась.

### Использованные способы отладки Microsoft Visual Studio

| № | Инструмент | Применение |
|---|---|---|
| 1 | **Точки останова (Breakpoints)** | Установлены на цикле `foreach` и на условии фильтрации, чтобы проверить, какие объекты попадают в выборку. |
| 2 | **Шаг с заходом — Step Into (F11)** | Выполнялся вход внутрь метода `FindGalaxies()` для проверки логики фильтрации. |
| 3 | **Окно автозапуска (Autos Window)** | Автоматически отображались переменные текущей и предыдущей строки: `galaxy`, `GalaxyType`, результат сравнения. |
| 4 | **Окно локальных переменных (Locals Window)** | Отслеживался список `filteredGalaxies` и текущий элемент `galaxy` на каждой итерации цикла. |
| 5 | **Подсказки по данным (Data Tips)** | При наведении курсора на объект `galaxy` раскрывались все его свойства прямо в редакторе. |
| 6 | **Немедленное окно (Immediate Window)** | Вычислялись выражения `galaxy.Name`, `string.Compare(...)` во время остановки для проверки гипотез об ошибке. |
| 7 | **Стек вызовов (Call Stack Window)** | Просмотрена цепочка вызовов от `Main` до `FindGalaxies` для понимания контекста выполнения. |
| 8 | **Окно точек останова (Breakpoints Window)** | Просмотрены все установленные точки останова и управление ими в одном месте. |

### XML-комментарии

Добавлены XML-комментарии к классу `Galaxy`, методам `FindGalaxies` и `Main` (см. файл `galaxies_Program.cs`).

## Задание 3 — Консольное приложение «Буквы»

**Источник:** [Microsoft Learn — Tutorial: Debug C# code](https://learn.microsoft.com/ru-ru/visualstudio/get-started/csharp/tutorial-debugger?view=vs-2022#create-a-project)  
**Ветвь Git:** `letters-debug`

### Описание задачи

Приложение строит строку из набора букв в цикле, а затем передаёт её в метод `SendMessage`. Ошибка состояла в том, что в цикле использовался неверный индекс массива (выход за границы), из-за чего бросалось исключение `IndexOutOfRangeException`.

### Использованные способы отладки Microsoft Visual Studio

| № | Инструмент | Применение |
|---|---|---|
| 1 | **Точки останова (Breakpoints)** | Установлена на строке `name += letters[i]` внутри цикла `for`, чтобы проверять значение индекса и элемента массива на каждой итерации. |
| 2 | **Шаг — Step Over (F10)** | Использовался для последовательного выполнения итераций цикла и наблюдения за накапливаемой строкой `name`. |
| 3 | **Шаг с заходом — Step Into (F11)** | Выполнялся заход внутрь метода `SendMessage()` для проверки полученного аргумента. |
| 4 | **Шаг с выходом — Step Out (Shift+F11)** | После проверки `SendMessage` использовался выход обратно в `Main`. |
| 5 | **Подсказки по данным (Data Tips)** | Раскрытие массива `letters` при наведении: просмотр всех элементов и их индексов, что помогло обнаружить выход за границы. |
| 6 | **Окно автозапуска (Autos Window)** | Автоматически отображались `i`, `name`, `letters[i]` — переменные текущей строки — на каждом шаге. |
| 7 | **Окно локальных переменных (Locals Window)** | Отслеживались все переменные метода `Main`: `letters`, `name`, `i`, `length`. |
| 8 | **Перезапуск отладки (Restart — Ctrl+Shift+F5)** | После внесения исправления применялся быстрый перезапуск без закрытия Visual Studio. |

## Общий вывод

В ходе выполнения практической работы были освоены следующие инструменты отладчика Microsoft Visual Studio:

- **Точки останова** (обычные, условные, временные)
- **Управление выполнением** (Step Into, Step Over, Step Out, Continue, Restart)
- **Окна наблюдения** (Locals, Autos, Watch, Immediate)
- **Подсказки по данным (Data Tips)** — быстрый просмотр значений прямо в редакторе
- **Стек вызовов (Call Stack)** — анализ цепочки вызовов при исключениях
- **Окно точек останова (Breakpoints Window)** — централизованное управление

Использование отладчика позволило точно локализовать каждую ошибку без «угадывания» и значительно ускорило процесс её исправления по сравнению с методом `Console.WriteLine`.
